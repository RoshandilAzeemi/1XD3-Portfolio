Entry 1 (January 18)

My experience of the course COMPSCI 1XD3 has been pretty fun so far. This is because it has been fun learning to build my own website / portfolio. Moreover, I also like making my website look nice and styling it as well.

Entry 2 (January 20)

Through the first part of this course, I have learned how to structure and style web pages using HTML and CSS, with a stronger focus on layout and responsive design. Updating my portfolio for this lab helped me better understand how spacing, navigation, and organization affect how professional a website looks. Overall, I am happy with the updated design because it feels cleaner and more structured than my previous version. There are still areas that could be improved, such as refining the color scheme and adding more content to better represent my work. I now feel more comfortable using CSS concepts like the box model and flexbox to control layout and design. As we finish the user interface unit, I am looking forward to learning how to make websites more interactive and dynamic in the remainder of the course.

Entry 3 (February 4)

I found it fairly easy to leverage my prior programming experience to learn client-side JavaScript, especially the logic and control flow, but how the DOM interacts with code was surprisingly different from what I was used to. The most challenging part has been debugging small issues caused by element selection, script loading order, and how user interactions trigger events rather than code running in a straight line.

Entry 4 (February 26)

Working with classes and objects has been the most challenging due to the concept being a little complex, while DOM manipulation feels the most intuitive since the visual results are immediate. I think classes are the most fundamental to a career in computer science because they form the backbone of scalable, object-oriented software design used in almost every major industry. Mastering these structures allows for the creation of reusable code.

Entry 5 (March 11)

I find server-side coding more rewarding because it provides the satisfaction of building the application's core logic and "brain" rather than just the surface visuals. The most important role of server-side scripting is to act as a secure gatekeeper for data and business logic, ensuring that sensitive operations remain protected from the user's browser. Meanwhile, the most important role of client-side scripting is to deliver a fast, interactive user experience by handling real-time interface updates without constant page reloads.

Entry 6 (March 31)

Reflecting on this course, the most significant shift was moving from static layouts to dynamic, stateful applications. Mastering PHP session management and AJAX were the clear highlights, these tools turned simple pages into interactive experiences, like the slot machine and guessing game. I can certainly see myself building a career in web development. The logic of the "backend" managing data and user states is particularly satisfying. In the future, I’d like to deepen my understanding of Security best practices (to protect user data) and explore modern frameworks like React to see how it streamlines the manual processes I learned here.
